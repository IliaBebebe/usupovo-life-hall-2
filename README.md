## 🚀 Быстрый старт

### Требования
- Node.js 20.x или выше
- npm или yarn

### Установка

1. **Клонируйте репозиторий** (или скачайте архив)

```bash
git clone <repository-url>
cd usupovo-life-hall
```

2. **Установите зависимости**

```bash
npm install
```

3. **Запустите сервер**

```bash
npm start
```

Для разработки (с автоперезагрузкой):
```bash
npm run dev
```

4. **Откройте сайт в браузере**

- 🏠 Главная страница: [http://localhost:3000](http://localhost:3000)
- ⚙️ Админ-панель: [http://localhost:3000/admin.html](http://localhost:3000/admin.html)
- ✅ Режим билетёра: [http://localhost:3000/verify.html](http://localhost:3000/verify.html)

## 🔌 API

### Основные маршруты

#### Мероприятия
- `GET /api/events` — список всех мероприятий
- `GET /api/events/:id` — детали конкретного мероприятия

#### Места
- `GET /api/seats/event/:eventId` — схема мест для мероприятия

#### Бронирование
- `POST /api/book` — создание бронирования
  ```json
  {
    "eventId": 1,
    "seatLabels": ["A1", "A2"],
    "customerName": "Иванов Иван",
    "customerEmail": "ivan@mail.ru",
    "customerPhone": "+79991234567",
    "discountCategoryId": 2,
    "promoCode": "SUMMER2024"
  }
  ```

#### Билеты
- `GET /api/ticket/:ticketId` — проверка билета по номеру
- `POST /api/ticket/:ticketId/use` — отметить билет как использованный

#### Промокоды
- `GET /api/promo/validate/:code` — проверка промокода
- `POST /api/promo/create` — создание промокода (админ)

#### Льготные категории
- `GET /api/discount-categories` — список всех категорий

#### Платежи
- `POST /api/create-payment` — создание платежа
- `POST /api/confirm-payment` — подтверждение оплаты

### Админские маршруты

#### Мероприятия
- `GET /api/admin/events` — все мероприятия (с продажами/выручкой)
- `POST /api/admin/events` — создать мероприятие
- `PUT /api/admin/events/:eventId` — редактировать мероприятие
- `DELETE /api/admin/events/:eventId` — удалить мероприятие

#### Места
- `GET /api/admin/events/:eventId/seats` — получить места мероприятия
- `POST /api/admin/events/:eventId/seats/bulk` — массовое создание мест
- `PUT /api/admin/seats/:seatId` — редактировать место
- `DELETE /api/admin/seats/:seatId` — удалить место

#### Бронирования
- `GET /api/admin/bookings` — все бронирования
- `DELETE /api/admin/bookings/:bookingId` — удалить бронирование (места автоматически освобождаются)
- `GET /api/admin/stats` — статистика сайта

#### Промокоды
- `GET /api/admin/promocodes` — список всех промокодов
- `POST /api/admin/promocodes` — создать промокод
- `PUT /api/admin/promocodes/:id` — редактировать промокод
- `DELETE /api/admin/promocodes/:id` — удалить промокод

#### Льготные категории
- `GET /api/admin/discount-categories` — список категорий
- `POST /api/admin/discount-categories` — создать категорию
- `PUT /api/admin/discount-categories/:id` — редактировать категорию
- `DELETE /api/admin/discount-categories/:id` — удалить категорию

#### Статистика посетителей
- `GET /api/admin/visitor-stats` — список всех сессий (с фильтрами по датам)
- `GET /api/admin/visitor-stats/aggregated` — агрегированная статистика (общая статистика, устройства, браузеры, ОС)
- `GET /api/admin/visitor-stats/chart?days=30` — данные для графика (уникальные посетители по дням)

#### Резервное копирование
- `GET /api/admin/export` — экспорт всех данных БД в JSON файл
  - Возвращает JSON файл со всеми таблицами базы данных
  - Файл автоматически скачивается с именем `usupovo-hall-backup-YYYY-MM-DD.json`
- `POST /api/admin/import` — импорт данных из JSON файла или по URL
  - Принимает JSON с данными для импорта
  - Поддерживает импорт как из локального файла, так и по URL ссылке
  - Автоматически восстанавливает связи между таблицами
  - Сохраняет информацию о последнем импорте
  ```json
  {
    "backupFilename": "usupovo-hall-backup-2025-01-15.json",
    "exportDate": "2025-01-15T10:00:00.000Z",
    "tables": {
      "events": [...],
      "seats": [...],
      "tickets": [...],
      ...
    }
  }
  ```
- `GET /api/admin/last-backup-info` — получение информации о последнем загруженном бэкапе
  - Возвращает имя файла, дату импорта и дату экспорта

## 🗄️ База данных

Используется SQLite3. База данных создается автоматически при первом запуске.

### Таблицы
- `events` — мероприятия
- `seats` — места в зале
- `tickets` — билеты
- `pending_bookings` — ожидающие оплаты бронирования
- `promocodes` — промокоды
- `discount_categories` — льготные категории
- `visitor_sessions` — детальная статистика посещений (IP, устройство, браузер, ОС, время входа/выхода, длительность, посещенные страницы)
- `settings` — настройки системы (тип билета, информация о последнем бэкапе и т.д.)

## 🛠️ Технологии

- **Backend:** Node.js + Express
- **База данных:** SQLite3
- **Frontend:** Vanilla JavaScript (ES6+)
- **Стили:** CSS3 с адаптивным дизайном
- **PWA:** Service Worker
- **QR-коды:** html5-qrcode библиотека
